# Example 07: Full PR Simulation (End-to-End)

## What This Tests

A complete end-to-end simulation of NitroGate running against a PR that contains multiple security issues across all scanners. This is the demo you'd show to hackathon judges.

## The Scenario

A developer opens a PR in `myorg/myrepo` that:
1. Adds an AWS key in a config file (secrets)
2. Adds `event-stream` as a dependency (compromised npm package)
3. Adds a CI workflow with script injection (workflow security)
4. Adds `lodash@4.17.20` (known CVE)
5. Adds a GPL-3.0 licensed dependency (license violation)

## How to Run

### Option A: Run Go unit tests (no GitHub needed)

```bash
cd /path/to/nitrogate
go test ./internal/... -v -count=1
```

### Option B: Run against a real GitHub PR (needs personal repo)

1. Create a GitHub org (e.g., `nitrogate-demo`)
2. Create a test repo with a `.nitrogate.json` policy
3. Push a PR with the test files from this directory
4. Run NitroGate as a GitHub Action

### Option C: Run the local simulation script

```bash
cd /path/to/nitrogate/examples/07-full-pr-simulation
./simulate.sh
```
